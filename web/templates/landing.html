How it works

Post-quantum by default

ML-KEM-768 key encapsulation with AES-256-GCM — protection against tomorrow's quantum attacks, on every transfer.

Sealed to the recipient

Files are encrypted to the recipient's public key. The server keeps only ciphertext — at rest, your data isn't readable.

Burn after download

Each file is deleted the instant the recipient downloads it. Nothing lingers on the server.

Accounts, 2FA & an API
